# Care Passport Workspace

WhatsApp voice companion / evidence-backed care passport for patients.

## Structure

```
backend/        - Python API + worker (care-passport service)
  admin/        - Next.js admin UI
docs/           - product thinking, research, pitch notes, and decisions
```

- `AGENTS.md` - working instructions for AI coding agents and collaborators.

## Working Principles

- Prioritize a demoable product over broad architecture.
- Keep notes concise and decision-oriented.
- Keep code scoped to the prototype path needed for the pitch.
- Do not place implementation files at the repository root unless they are project-level config.
- Capture major product decisions in `docs/` as they are made.
