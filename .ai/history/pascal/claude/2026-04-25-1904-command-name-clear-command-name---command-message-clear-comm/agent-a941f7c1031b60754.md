## Product context (so you can make judgment calls)

Care-passport is a WhatsApp voice companion that builds an evidence-backed knowledge graph about a patient. Each patient has one long-running Temporal workflow. Inbound WhatsApp messages arrive as **signals** into the per-patient workflow. The workflow calls activities to (a) extract structured facts from the message, (b) write them to a Graphiti knowledge graph keyed by `group_id=patient_id`, (c) retrieve context, (d) generate a reply via Claude, (e) send the reply back. A FastAPI service sits alongside to query the same Graphiti store for completeness/staleness reports.

The user (Pascal) will define the patient ontology (Pydantic entity types) in a follow-up task. **Do NOT design entity types yourself** — just leave a clearly-marked extension point with one toy entity (e.g. `Symptom`) so the wiring is testable.

- **uv** for package management — `uv init`, `uv add`, `pyproject.toml`. Python 3.12.
